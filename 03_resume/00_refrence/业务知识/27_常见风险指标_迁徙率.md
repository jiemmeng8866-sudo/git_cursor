# 迁徙率

## 1 业务含义

***迁徙率（同称：净流量滚动比例法 / Flow Rate Analysis）：***处于某一逾期状态的客户转移至下一个逾期状态的比例；其中，逾期期数（M）指根据逾期天数来划分不同的逾期状态：

| 逾期状态 | 逾期介绍                    |
| -------- | --------------------------- |
| M0       | 逾期天数为0（未发生逾期）   |
| M1       | 逾期天数为1-30天（含30天）  |
| M2       | 逾期天数为31-60天（含60天） |
| M3       | 逾期天数为61-90天（含90天） |
| M3+      | 逾期天数为61天以上          |



![img](C:\Users\m8705\AppData\Roaming\Typora\typora-user-images\image-20260214131004944.png)

➤ 迁徙率体现了客户贷款账户在整个生命周期内的变化轨迹，迁徙率高即资产回催效果不佳或用户还款意愿较低，可预测不同逾期阶段的未来坏账损失；➤ ***迁徙率口径：一般分为客户/余额维度，需剔除万一贷和逾期不满60天但五级分类为******不良******的借据***【基于部分纾困行为，借据表中存在一部分逾期未满60天但五级分类为不良的借据：其旧借据在未结清前已逾期，发生借新还旧生成的新借据若五级分类为不良，其借据一定为坏借据（继续逾期），其本质相当于旧借据的基础上继续逾期，逾期状态为M3+】➤ **参考资料：**[风险管理常用指标分享](https://learning.weoa.com/bbs/#/postdetail/dfd1f1ed-29cc-4e50-8fd8-62df5776cee3)

## 2 迁徙率

***Step1：计算同笔借据在未来30天后的******逾期******天数及剔除万一贷和逾期不满60天但五级分类为******不良******的借据；***

```Plaintext
--可直接用brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds_extend表来替代brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds借据表
--brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds_extend表含overduedays_after30 字段，可无需再计算
select duebill_serialno,overduedays,balance,ccif,ds,lead(overduedays,30,null) over (partition by duebill_serialno order by ds) as overduedays_after30 --30天后的逾期天数
from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
where ds >= '20230101' and duestatus in ('0','1','2','3') and nvl(is_oneyidai_vintage,0)= 0 --剔除万一贷
        and not (coalesce(cast(overduedays as bigint),0) < 60 and classifyresult in ('03','04','05')) --剔除逾期不满60天但五级分类为不良的借据
```

***Step2：以观察时点来计算M0、M1、M0-M1的在贷户数/余额；***

```Plaintext
--M0、M1、M0-M1在贷余额
,sum(case when coalesce(cast(overduedays as bigint),0) = 0 then balance else 0 end) as balance_m0,--M0金额
,sum(case when coalesce(cast(overduedays as bigint),0) > 0 and coalesce(cast(overduedays as bigint),0) <= 30 then balance else 0 end) as balance_m1,--M1逾期金额
,sum(case when coalesce(cast(overduedays as bigint),0) = 0 and coalesce(cast(overduedays_after30 as bigint),0) > 0 and coalesce(cast(overduedays_after30 as bigint),0) <= 30 then balance else 0 end) as balance_m0_m1,--M0-M1逾期金额
--M0、M1、M0-M1在贷户数
,count(distinct case when coalesce(cast(overduedays as bigint),0) = 0 then ccif else null end) as cust_m0,--M0客户数
,count(distinct case when coalesce(cast(overduedays as bigint),0) > 0 and coalesce(cast(overduedays as bigint),0) <= 30 then ccif else null end) as cust_m1,--M1客户数
,count(distinct case when coalesce(cast(overduedays as bigint),0) = 0 and coalesce(cast(overduedays_after30 as bigint),0) > 0 and coalesce(cast(overduedays_after30 as bigint),0) <= 30 then ccif else null end) as cust_m0_m1,--M0-M1逾期客户数
```

***Step3：以观察时点来计算M0-M1余额迁徙率/户数迁徙率***

```Plaintext
--M0-M1余额迁徙率/户数迁徙率
balance_m0_m1/balance_m0 as amt_M0_M1   --M0-M1余额迁徙率
cust_m0_m1/cust_m0 as cnt_M0_M1         --M0-M1户数迁徙率
```

## 3 参考代码

```SQL
select observe_month,balance_m0,balance_m1,balance_m2,balance_m3,balance_m3_up,balance_m0_m1,balance_m1_m2,balance_m2_m3,balance_m0_m3,
        cust_m0,cust_m1,cust_m2,cust_m3,cust_m3_up,cust_m0_m1,cust_m1_m2,cust_m2_m3,cust_m0_m3,
        balance_m0_m1/balance_m0 as amt_M0_M1, --M0转为M1的余额迁移率
        balance_m1_m2/balance_m1 as amt_M1_M2, --M1转为M2的余额迁移率
        balance_m2_m3/balance_m2 as amt_M2_M3, --M2转为M3的余额迁移率
        cust_m0_m1/cust_m0       as cnt_M0_M1, --M0转为M1的客户迁移率
        cust_m1_m2/cust_m1       as cnt_M1_M2, --M1转为M2的客户迁移率
        cust_m2_m3/cust_m2       as cnt_M2_M3  --M2转为M3的客户迁移率
from(
    select 
    substr(ds,1,6) as observe_month,
    sum(case when coalesce(cast(overduedays as bigint),0) = 0 then balance else 0 end) as balance_m0,--M0金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 0 and coalesce(cast(overduedays as bigint),0) <= 30 then balance else 0 end) as balance_m1,--M1逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 30 and coalesce(cast(overduedays as bigint),0) <= 60 then balance else 0 end) as balance_m2,--M2逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 60 and coalesce(cast(overduedays as bigint),0) <= 90 then balance else 0 end) as balance_m3,--M3逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 90 then balance else 0 end) as balance_m3_up, --M3+逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) = 0 and coalesce(cast(overduedays_after30 as bigint),0) > 0 and coalesce(cast(overduedays_after30 as bigint),0) <= 30 then balance else 0 end) as balance_m0_m1,--M0-M1逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 0 and coalesce(cast(overduedays as bigint),0) <= 30 and coalesce(cast(overduedays_after30 as bigint),0) > 30 and coalesce(cast(overduedays_after30 as bigint),0) <= 60 then balance else 0 end) as balance_m1_m2,--M1-M2逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) > 30 and coalesce(cast(overduedays as bigint),0) <= 60 and coalesce(cast(overduedays_after30 as bigint),0) > 60 and coalesce(cast(overduedays_after30 as bigint),0) <= 90 then balance else 0 end) as balance_m2_m3,--M2-M3逾期金额
    sum(case when coalesce(cast(overduedays as bigint),0) = 0 and coalesce(cast(overduedays_after30 as bigint),0) > 60 and coalesce(cast(overduedays_after30 as bigint),0) <= 90 then balance else 0 end) as balance_m0_m3,--M0-M3逾期金额
    count(distinct case when coalesce(cast(overduedays as bigint),0) = 0 then ccif else null end) as cust_m0,--M0客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 0 and coalesce(cast(overduedays as bigint),0) <= 30 then ccif else null end) as cust_m1,--M1客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 30 and coalesce(cast(overduedays as bigint),0) <= 60 then ccif else null end) as cust_m2,--M2客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 60 and coalesce(cast(overduedays as bigint),0) <= 90 then ccif else null end) as cust_m3,--M3客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 90 then ccif else null end) as cust_m3_up, --M3+逾期客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) = 0  and coalesce(cast(overduedays_after30 as bigint),0) > 0 and coalesce(cast(overduedays_after30 as bigint),0) <= 30 then ccif else null end) as cust_m0_m1,--M0-M1逾期客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 0  and coalesce(cast(overduedays as bigint),0) <= 30 and coalesce(cast(overduedays_after30 as bigint),0) > 30 and coalesce(cast(overduedays_after30 as bigint),0) <= 60 then ccif else null end) as cust_m1_m2,--M1-M2逾期客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) > 30 and coalesce(cast(overduedays as bigint),0) <= 60 and coalesce(cast(overduedays_after30 as bigint),0) > 60 and coalesce(cast(overduedays_after30 as bigint),0) <= 90 then ccif else null end) as cust_m2_m3,--M2-M3逾期客户数
    count(distinct case when coalesce(cast(overduedays as bigint),0) = 0 and coalesce(cast(overduedays_after30 as bigint),0) > 60 and coalesce(cast(overduedays_after30 as bigint),0) <= 90 then ccif else null end) as cust_m0_m3--M0-M3逾期客户数
    from(
        select duebill_serialno,overduedays,balance,ccif,ds,
            lead(overduedays,30,null) over (partition by duebill_serialno order by ds) as overduedays_after30 --30天后的逾期天数
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds >= '20230101'
            and nvl(is_oneyidai_vintage,0)= 0 --剔除万一贷
            and duestatus in ('0','1','2','3')
            and not (coalesce(cast(overduedays as bigint),0) < 60 and classifyresult in ('03','04','05')) --剔除逾期不满60天但五级分类为不良的借据
    )
    group by substr(ds,1,6)
)
```
