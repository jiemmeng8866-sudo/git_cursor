# 逾期生成率

## 1 业务含义

**逾期生成率：**反映了新增逾期金额（客户）【分子】占日均正常在贷M0余额（客户）【分母】的比例变化情况；

**➤【分母】：T月日均正常在贷（未逾期）M0余额;**

**➤【分子】：T月新增逾期余额：**若该笔借据在T月月末是正常未逾期的状态，但到T+1月月末却是逾期的状态，则此借据属于新增逾期的借据，则新增逾期余额【存量正常_当前M1逾期】为T+1月月末时点的借据余额；

➤ 举例：

   9月新增逾期余额【分子】：为9月月末时点未逾期的在贷借据到10月月末时点变成逾期的部分借据的余额【此处取得是10月月末时点的余额】；       

   9月日均正常在贷M0余额【分母】：为9月内的每天的正常在贷未逾期的借据的总余额的加总/9月的天数（30天）；       9月逾期生成率 = 9月新增逾期余额 / 9月日均正常在贷M0余额；

## 2 逾期生成率

逾期生成率的计算与另类风险成本率类似，需进行借据重组、合理映射等步骤：

***Step1 - 借据重组：***将所有借据进行还原重组即发生过借新还旧的新借据会匹配至旧借据，旧借据仍为旧借据，未发生借新还旧的借据保持一致；注：此处的余额无需进行不良资产转让还原！

***Step2 - 合理映射：***需将新借据当前状态映射至旧借据，再以旧借据进行关联；

***Step3 - 计算T月的新增逾期金额、新增逾期户数、日均在贷正常M0金额、日均在贷正常M0客户数；***

☑ 新增逾期金额：  【1】LUM口径：原字段；  【2】表内口径：视同核销、零元回购、担保代偿、保险代偿、小微银银2.1均纳入新增逾期金额统计；☑ 日均在贷正常余额：  【1】LUM口径：原字段；  【2】表内口径：视同核销、零元回购、小微银银2.1、保险代偿、担保代偿转让均纳入核销后日均在贷正常余额统计；☑ 客群、产品、策略、行业、省份、企金整体等维度信息以旧借据信息为主、表内标签取得是月末时点的数据；表内判断逻辑：coalesce(is_outside_table_package,0)=0 and source_tbl in ('mcfcm','cfcm','ymcfcm') 

**➤** ***中间表-数据结构-注释：***

（1）分子和分母存在口径不一致的情况：另类风险成本率与逾期生成率的日均在贷正常余额没有核销前后的概念，而是LUM口径下视同核销和零元回购的借据不影响日均正常在贷的计算余额；（2）针对1月份的逾期生成率，由于2月的天数相对于其他月份少，同时一般入催是在第30天进行入催，因此为了和其他月份保持一致，2月份的逾期生成率作出如下调整：【1】新增逾期金额：1月月末的正常在贷的借据在2月月末时点为逾期的状态，且延长至3月3日也处于逾期状态的逾期金额【2】日均在贷余额：1月的日均正常在贷的金额

### ➤ 中间表-使用方法

```Plaintext
--举例：当月新增逾期金额：sum(balance_yq_m_1)
--举例：逾期生成率-日均贷款：sum(balance_yq_m0)/count(distinct watching_ds)
select ds_month,
       sum(balance_yq_m_1)/100000000 as yuqi_add_lum,
       sum(balance_yq_m0)/count(distinct watching_ds) as avg_balance_lum,
       sum(balance_yq_m_1)/(sum(balance_yq_m0)/count(distinct watching_ds)) as yuqi_rate_lum,
       sum(balance_bn_yq_m_1)/100000000 as yuqi_add_bn,
       sum(balance_bn_yq_m0)/count(distinct watching_ds) as avg_balance_bn,
       sum(balance_bn_yq_m_1)/(sum(balance_bn_yq_m0)/count(distinct watching_ds)) as yuqi_rate_bn     
from brm_ent_dm_work.dwd_fxcb_yqlv_by_duebill_ds_extend
where ds = '20231130'-- 添加行业/客群/企金整体等筛选条件
group by ds_month
order by ds_month
```

## 4 全流程计算-参考代码

```SQL
--客户维度/金额维度-逾期生成率
-- Step1：将所有借据进行还原重组至origin_duebill_serialno_traceback字段，即发生过借新还旧的新借据会匹配至旧借据，旧借据仍为旧借据，未发生借新还旧的借据保持一致：
with tmp as(
    select 
        t.*,,case when coalesce(is_bxdc,0) = 1 then we_balance_bxdc
                  when coalesce(is_dbdc,0) = 1 then we_balance_dbdc
             else balance end as balance_lum,
        row_number() over(partition by coalesce(a.origin_duebill_serialno,t.duebill_serialno),ds order by putout_date asc) as rn_asc,
        row_number() over(partition by coalesce(a.origin_duebill_serialno,t.duebill_serialno),ds order by putout_date desc) as rn_desc,
        coalesce(a.origin_duebill_serialno,t.duebill_serialno) as origin_duebill_serialno_traceback --借新还旧还原成老借据
    from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds t 
    left join( --借新还旧重组
        select t1.new_duebill_serialno_jxhj,
        coalesce(t5.old_duebill_serialno_jxhj,t4.old_duebill_serialno_jxhj,
                t3.old_duebill_serialno_jxhj,t2.old_duebill_serialno_jxhj,
                t1.old_duebill_serialno_jxhj) as origin_duebill_serialno
        from(select * from brm_ent_dm_work.dwd_jxhj_by_old_duebill where ds = '${run_date}')t1
        left join(select * from brm_ent_dm_work.dwd_jxhj_by_old_duebill where ds = '${run_date}')t2 on t1.old_duebill_serialno_jxhj = t2.new_duebill_serialno_jxhj
        left join(select * from brm_ent_dm_work.dwd_jxhj_by_old_duebill where ds = '${run_date}')t3 on t2.old_duebill_serialno_jxhj = t3.new_duebill_serialno_jxhj
        left join(select * from brm_ent_dm_work.dwd_jxhj_by_old_duebill where ds = '${run_date}')t4 on t3.old_duebill_serialno_jxhj = t4.new_duebill_serialno_jxhj
        left join(select * from brm_ent_dm_work.dwd_jxhj_by_old_duebill where ds = '${run_date}')t5 on t4.old_duebill_serialno_jxhj = t5.new_duebill_serialno_jxhj
    )a on t.duebill_serialno = a.new_duebill_serialno_jxhj
    where t.duestatus in ('0','1','2','3') and t.ds > '20230601' and substr(date_add(from_unixtime(unix_timestamp(ds,'yyyyMMdd'),'yyyy-MM-dd'),1),9,2) = '01' --月末
),temp as(
    select * from(
        select t.duebill_serialno as duebill_serialno_origin,a.classifyresult,a.balance_lum,t.ccif,t.ds,a.overduedays,row_number() over(partition by a.duebill_serialno,t.ds order by t.duebill_serialno desc) as rn
            from (select * from tmp where rn_asc = 1) t
            inner join(
                    select * from tmp where rn_desc = 1
                )a on a.origin_duebill_serialno_traceback = t.origin_duebill_serialno_traceback and t.ds = a.ds
    )t
    where rn = 1
)
select t.ds,
        yuqi_add_cnt/avg_cnt as yuqi_cnt_rate, --客户维度逾期生成率
        yuqi_add_amt/avg_amt as yuqi_amt_rate  --金钱维度逾期生成率
from(
    select t.ds,
    sum(case when t_m.classifyresult in ('01') and t.classifyresult in ('01','02') and t.overduedays > 0 and t.overduedays <= 30 then t.balance_lum else 0 end) as yuqi_add_amt, --核销前新增逾期金额
    count(distinct case when t_m.classifyresult in ('01') and t.classifyresult in ('01','02') and t.overduedays > 0 and t.overduedays <= 30 then t.ccif else null end) as yuqi_add_cnt --核销前新增逾期户数
    from temp t_m
    left join temp t on t.duebill_serialno_origin = t_m.duebill_serialno_origin and substr(t.ds,1,6) = replace(substr(date_add(from_unixtime(unix_timestamp(t_m.ds,'yyyyMMdd'),'yyyy-MM-dd'),1),1,7),'-','')
    group by t.ds
)t 
left join( --Step2 日均日均正常金额 avg_amt,日均正常客户数 avg_cnt
    select substr(ds,1,6) as ds_month,avg(cnt) as avg_cnt,avg(amt) as avg_amt
    from(
        select ds,
            count(distinct case when duestatus in ('0') and classifyresult in ('01') and overduedays = 0 then ccif else null end) as cnt, --日均在贷正常M0金额
            sum(case when duestatus in ('0') and classifyresult in ('01') and overduedays = 0 then balance else 0 end) as amt --日均在贷正常M0金额
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds >= '20230601'
        group by ds
    )
    group by substr(ds,1,6)
)t1 on substr(t.ds,1,6)= t1.ds_month
```
