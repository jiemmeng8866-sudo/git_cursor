## 一、一条线：信贷全生命周期（背顺序）

**进件 → 核额 → 授信 → 提款 → 放款 → 还款 → 催收**

| 环节 | 一句话 |
|------|--------|
| 进件 | 渠道进产品、实名与授权、签协议后才可申请核额 |
| 核额 | 限额审查、更新可贷额度；可能触发反欺诈（如视频双录） |
| 授信 | 审批通过后授予信用额度 |
| 提款 | 签约/人脸等 → **风险系统拉提款引擎** → 是否下发额度 |
| 放款 | 组合系统下发贷款 |
| 贷后 | 监测还款行为：正常、提前结清、逾期、延期等 |
| 催收 | 分层触达；严重则冻额、征信；末端涉及不良处置（转让、ABS 等） |

**贷前技术链（口述版）：** 组合系统进件 → 风险系统 **路由：拉数 → 清洗 → 生产变量 → 引擎/规则 → 回传**；同时 **统一授信台** 校验；返回 **主标尺（统一风险刻度）+ 定价/额度方案**。

## 二、消费贷 vs 经营贷（对比记）

| | 消费贷 | 经营贷 |
|---|--------|--------|
| 用途 | 日常/大宗消费 | 企业经营周转 |
| 典型特征 | 额度偏低、利率偏高、期限偏短 | 常要担保/抵押，额度与期限相对更灵活 |
| 例子 | 微粒贷、花呗、白条等 | 微业贷、网商贷、金企贷等 |

## 三、企金条线产品（记「主菜 + 变体」）

- **微业贷：** 企业信用 + 企业主信用、可循环、线上小微流贷（文档中的核心对公小微产品）。
- **科创贷：** 国高/专利/软著/小巨人等科创中小微企业流贷。
- **订货贷 / 供货贷：** 供应链场景；**订货**偏经销商/上游订货发票；**供货**偏下游，近 12 个月发票交易；改革后对应 **产业贷-上游 / 下游**。
- **微众易贷：** 有房基础的个体户/小微主，特定期数无抵押经营贷。

## 四、额度体系 V1 → V2 → V3（三句话）

1. **V1：** 各产品 **固定额度切分**，与申请顺序相关；总额不超模型总额度。
2. **V2：** **按占用动态切分**，更灵活；总额仍不超模型总额度。
3. **V3：** 在 V2 上加 **产品组合约束**：互斥/合并/非合并；互斥或合并组合下 **核额流水表通常只有一个产品额度有效**；下发顺序文档示例：**KCD > DHD > GHD > WYD**。

## 五、资产口径「五块拼图」（面试高频）

| 名词 | 记忆锚 |
|------|--------|
| **LUM（管理贷款）** | 我行 **持续管理** 的贷款：表内 + 联贷他行份额 + 技术性出表 + 通道费等合作形态；覆盖贷前中后与存续管理 |
| **表内** | **未出表** 的我行份额资产 |
| **分流** | **小微银银 2.0 / 2.1** 中划归 **我行自营额度** 的部分（与「全合作行出资的 2.0 整笔在 xwyy」区分记忆） |
| **出表 ABS** | 证券化转让后 **非表内**、仍属 LUM 管理范畴的典型形态 |
| **出表 ABN** | 信托 ABN：**私募机构出资**，我行多 **通道**；可多次转让、借据类型复杂 |

**小微银银（文档口径）：**

- **2.0：** 额度分流、合作行模式；借据在 **企贷云**；`source_tbl = 'xwyy'`（文档示例字段）。
- **2.1：** 借据级联合贷，**我行与合作行按比例**；`is_xwyy_21`；主子借据后缀示例 **Y / YWB / YUN**。
- **3.0：** 2.0 与 2.1 形态结合，**企贷云**；`is_xwyy_30`。

## 六、处置与特殊产品（各记 1 句）

| 概念 | 一句话 |
|------|--------|
| **保险代偿** | 逾期达条件后 **保险公司** 按约定赔付；企金文档示例 **80%** 代偿部分可 **出表**，**20% 仍表内**；**催收主体仍为我行** |
| **担保代偿** | **担保机构** 代偿；文档示例 **全额**；**非 LUM**；催收主体为 **担保公司** |
| **零元回购** | **0 元**买回已售表外包（多针对 **逾期超 60 天** 难回收资产）；监管报送上 **可不纳入 LUM**，分析仍可能要全量视角 |
| **折价回购** | 按本金余额 **折扣** 买回；弄清 **原值 / 折价 / 净值** 关系即可口述 |
| **锁期** | 选定方案后 **锁定期内不可提前还款**；影响 **余额留存率**；存在客诉提前结清（未必有表） |
| **万一贷** | 可配置 **万分之一、万分之二** 等利率的特殊套餐 |
| **纾困** | 借款人财务恶化时 **调整还款条款** 的方案 |
| **不良资产转让 vs 自营债权转让** | 均为 **表内→表外**；不良为 **不良认定资产** 打包转让；自营债权转让偏 **债权盘活**；转让成功借据上 **`is_outside_table_package`** 等标签变化（细节以表为准） |

## 七、ABS vs ABN（对照记）

| | ABS | ABN（信托型为主） |
|---|-----|-------------------|
| 本质 | 信贷资产证券化，盘活表内贷款 | 银行间市场结构化票据，常引入信托 SPV |
| 我行角色 | 证券化后多 **管理人 + 催收等服务** | 常 **通道**，底层私募出资 |
| 记忆句 | 「表内贷款 → 证券 → 投资者」 | 「信托 + 破产隔离 + 出表融资」 |

**证券化 vs 信贷（文档金句）：** 信贷是用 **现在** 资产担保解决现在问题；证券化是用 **未来现金流** 担保解决现在问题。

**pre-ABN vs ABN：** pre 为 **预备池、未封包转让**；ABN 已封包转让路径（字段 `acct_loan_fvociflag` 等区分，面试说概念即可）。

## 八、主标尺（统一风险刻度）

- **为什么：** 多模型 **分数不可比**，需 **统一标尺** 做客户级风险对齐。
- **怎么做：** 以 **不良率（坏样本率）** 为桥梁，把不同模型分映射到 **同一套等级**。
- **等级：** 文档列 **28 级**（3A+ … D），用途：**头部精细化、尾部聚合管理**。
- **校准手段：** 分箱、逻辑回归、保序回归等；实务要考 **风险波动 / 前瞻性系数** 与 **评级 vs 实际表现监控**。

## 九、引擎（概念题）

**引擎 = 核额/提款等环节跑模型与规则，有入参、出参（8 位大写 modelID 等）。**

- **核额：** 如通用核额 **ACOMCRSS**、小微旁路 **SHUICRSS/QHUICRSS**、供货 **AGHDCRSS** 等（不必全背，知分类即可）。
- **提款：** **AKCDCRSS**、产业 **TCYDCRSS**、订货 **TDHLCRSS** 等。
- **贷后/预警：** **AWYDDH** 等预警评分与处置模型链。

## 十、风险指标「一张表」

**维度：** 客户 / 金额 × 发放、首授、首借等 **观察锚时点** × **LUM / 表内 / 核销前/后** 等口径。

**高频指标口诀：**

- **风险成本率：** 时段 **新增不良** / **核销后日均在贷**（又称新增不良率思路）。
- **Vintage：** **放款月为桶**，**MOB=观察月−放款月**，看随账龄 **逾期/不良** 曲线是否稳定、有无异常放款月。
- **迁徙率：** 本期逾期状态 → 下期 **滚动** 到更差档的比例；M0/M1/M2…（文档：**M1=1–30 天** 等）。
- **入催 / 治愈 / 还款率：** **入催**=还日未还进入催收；**治愈**=入催后一段时间内还清应还；**入催率**≈本月入催余额/上月末 M0；**还款率**可拆为「按时 1−入催率」+「入催×治愈率」思路。

**不良资产转让还原（背「为什么」）：** 转让后借据余额/状态 **突变**，若不算还原，不良与 vintage 会 **失真**；故 **风险成本率、另类风险成本率、发放时点不良 vintage、m1→m2、m1/m2 转不良、回收率** 等常要做还原；文档写明 **微众易贷** 可不做的场景。

## 十一、五级分类与「不良」（监管口径题）

- 五类：**正常、关注、次级、可疑、损失**；**后三类 = 不良**。
- 逾期与分类挂钩（文档要点）：逾期后 **至少关注**；**90/270/360 天** 与 **次级/可疑/损失** 的「至少」分档（具体数字面试说趋势即可）。
- 处置手段：**清收、重组、核销、债权转让、不良 ABS** 等。

## 十二、数据架构（数据岗/风控岗常问）

| 层 | 要点 |
|----|------|
| **业务系统** | 仅存 **最新态**，一般不做历史分区 |
| **BDP** | 科技大数据集群；多 **未脱敏**，权限严 |
| **BDAP** | 业务分析平台；**贴源脱敏数据 + 集市/自助表** |

**系统后缀：** **ydcn** 个人、**bdcn** 企业、无后缀 **cdcn**（老）；集市 snap 常 **合并去重** 保证连续性。

**增量 vs 全量分区：** 全量 = 截至当日 **全状态快照**；增量 = **新增分区追加**；可结合 **分区体积、时间跨度、量随时间走势** 判断。

**常用库名记忆：** `brm_wyd_ods_mask` 企风 ODS、`cfd_wyd_ods_mask` 企金 ODS、`brm_ent_dm_work` 部门加工/集市。

## 十三、分离式保函 / 票据 / 征信 / 税银（有追问再展开）

- **票据：** 银承、商承、贴现、风控引擎 **ACPT/PTTX/STTX** 等（详见 `16_...`）。

- **税银报告：** 纳税等级、申报收入、违法违章等交叉验证经营实质（详见 `22_...`）。
