0 · 岗位映射：中台数仓在风控里做什么职位视角

风控中台数仓的核心价值，是把可审计、可复用、口径一致的数据资产，供给授信策略、贷后监控、资产质量与监管报送。你掌握的「核额流水 / 借据 / 额度树 / 引擎出入参」等主题，对应的就是事件明细层 + 风险主题层 + 指标服务层的建设与治理。

  • 贴源同步：业务系统 → BDP（全量/增量）；权限与脱敏边界清晰。
  • 风险明细：借据粒度、客户粒度、合同与额度树——支撑 Vintage、迁徙、不良生成。
  • 口径治理：LUM / 表内 / 出表 / 代偿 / 转让前后状态——避免「余额跳变」导致指标失真。
  • 服务输出：监管与内部管理报表、策略回溯、预警日批/月批模型入参。

1 · 银行风控规范（行业通用框架）记忆锚

面试时常用「监管语言 + 你方落地」各说一半。下列为商业银行普遍遵循的框架，可与你们内部制度对照，不替代具体行内文件。

风控中台数仓主要服务二道防线，并与一道（业务系统源数据）、三道（审计追溯）对齐。

1.1 数据治理与风险数据（面试高频词）

  • 唯一可信源：同一指标只有一个「主编码规则」，其他层引用而非重复计算。
  • 全链路血缘：从 ODS 字段 → 清洗规则 → 主题表 → 报表单元格可追溯。
  • 口径文档化：LUM / 表内 / 核销后余额等必须写清「入表/出表/代偿/转让」边界。
  • 分级授权与脱敏：生产敏感区（BDP）与分析区（BDAP）隔离是常态。

1.2 资产质量与分类（监管对齐）

五级分类：正常、关注、次级、可疑、损失；后三类为不良。逾期天数与分类「至少」挂钩（如逾期超 90 天至少次级等——以最新监管办法与行内细则为准）。数仓侧要保证分类结果时点快照与借据余额在同一观察日可 join。

4 · 贷前：获客 → 核额授信业务+数

业务：进件（渠道、实名、授权、协议）→ 核额（可能双录）→ 授信。风险系统执行：拉数 → 清洗 → 变量 → 引擎/规则；统一授信台校验；输出主标尺与额度/定价方案。

数仓：核额流水、申请事件、策略命中、模型分与主标尺映射表；注意多产品核额与V3 互斥/合并下「流水多条但仅一产品有效额度」等复杂形态。

风控规范挂钩：客户身份识别（KYC）、反洗钱与欺诈监测、授信集中度与限额——数仓需提供「客户—关联人—集团」可扩展关联键（以行内主数据为准）。

5 · 贷中：提款准入 → 放款引擎+表

业务：签约/人脸 → 提款准入 → 组合放款。数据：提款引擎产生变量出入参，用于通过率、提款分布、实时/准实时策略分析。

引擎记忆：不必背全量 modelID；分类回答「核额类 / 提款类 / 合并旁路 / 预警贷后」即可，并举例 ACOMCRSS、AKCDCRSS、AWYDDH 等（以你文档为准）。

6 · 贷后：监测 → 催收 → 不良处置资产流转

还款形态：正常、提前结清、逾期、延期等。锁期影响留存率；纾困可能产生「逾期天数与五级分类不一致」等特例，指标文档要求迁徙率等剔除部分借据。

7 · 指标体系：分子分母 × 观察锚 × 是否还原中台核心

前「新增不良」（含新发生 − 回退 + 核销处置变化等新口径要素） 分母：观察期内核销后日均在贷余额 Vintage：放款月为 cohort，MOB = 观察月 − 放款月；看曲线是否稳定、有无异常放款月 入催/治愈：入催率 ≈ 本月入催 / 上月末 M0；还款率 = 按时 + 催回
不良资产转让还原：转让后借据余额/状态跳变，不做还原则风险成本率、Vintage、m1→m2 等会失真；文档注明微众易贷部分场景可不还原。
维度说明
客户 / 金额同一指标两套结果可能都要支持
发放 / 首授 / 首借观察锚不同，分母时间窗不同
LUM / 表内 / 核销前/后报送口径 vs 管理口径对齐

8 · 主题表与统一授信（你方知识库要点）

  • 借据表：余额、逾期天数、五级分类、source_tbl、银银 2.1/3.0 子借据字段、出表与代偿标签。
  • 核额流水：首次/定期核额、多产品、model_prod_type 等策略口径。
  • 统一授信额度树：总额度、产品额度、互斥合并关系——与 V3 额度逻辑一致表述。
  • 主标尺结果表：多模型分数映射到统一等级，服务排序与限额。

具体表名与 SQL 以原始 Markdown 与当期口径为准；本页不展开字段级取数。

9 · 面试口述模板（风控数仓岗 45 秒）可直接背

「我熟悉信贷贷前—贷中—贷后全链路在数据上的落地：贴源同步与脱敏边界、借据/合同/还款计划等明细层建设、核额与提款引擎出入参主题沉淀，以及LUM/表内/出表/代偿/转让等多口径治理。指标侧做过或理解风险成本率、Vintage、迁徙、入催治愈，清楚不良资产转让还原对不良趋势与 MOB 曲线的影响。工程上重视血缘、口径文档、调度单向无环，对接监管与内部风险报表的一致性。」

追问准备清单

  • 「表内余额」与「LUM 余额」差在哪里？
  • 转让交割日附近，借据表余额突变如何做指标？
  • 全量分区与增量分区如何选分区键、如何做拉链/快照？
  • 策略回溯：当时生效的规则版本与特征如何存档？
