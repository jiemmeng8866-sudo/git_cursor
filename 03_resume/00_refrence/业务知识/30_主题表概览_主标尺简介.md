## 1、主标尺简介

**背景：**企业风险部存在较多模型，不同模型的相同预测结果基于模型的评分范围和标准不一致具有不可比性，同时模型的预测结果在不同建模场景和风控场景下的解释并不代表真实的违约概率，因此考虑到模型分数的排序性，模型训练的客群样本应具有较好的区分度，不同维度的客群风险应具有可比性，因此需构建客户层面统一的风险评价体系（模型分数的统一对齐）；

**定义：**主标尺是我行针对不同模型分数不可比性等问题所构建的统一风险测量体系，主标尺3.0共有28个等级；相关主标尺3.0模型：企业主2.0模型、发票贷贷前模型、科创专属模型、回捞模型等贷前模型及贷后预警监控模型；

**主标尺等级：**3A+(最优质), 3A, 3A-, 2A+, 2A, 2A-, A+, A, A-, 3B+, 3B, 3B-, 2B+, 2B, 2B-, B+, B, B-, 3C+, 3C, 3C-, 2C+, 2C, 2C-, C+, C, C-, D(最差)等28个级别；

**映射方案：**以不良率(坏样本率)为建立不同模型评分与标尺级别的映射关系，则客户可在不同模型下的评分下通过不良率映射关系到同一主标尺；

**应用：**主标尺的级别细化有助于对头部（优质）客户进行精细化管理、对尾部客户进行聚合管理；分析人员可将客户分成不同主标尺等级的客群，观察客群质量变化规律；

**表加工流程：**先由模型科室提供含产品贷前核额流水粒度的贷前主模型（含首次和非首次）和定期核额策略的主标尺信息【含加压前分数、加压后分数、加压后主标尺】，再由各贷前科室跟进提供各产品对应流水最终评级及来源，最后由集市合并加工（若加压分数为空，由非加压分数补全）；

## 3、常见字段介绍

**01 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.overduedays_max：用户最大逾期天数**

**说明：**来源集市借据表brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds，取用户max(overduedays)

**02 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.dhrh_score： 贷后融合分。**

**说明：**贷后融合分，来源brm_ent_dm_work.snap_marm_model_record的v3010314出参，model_id=ADHRCRSS。

**03 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.ln_dhrh_score：贷后融合分对数化。log(dhrh_score/(1-dhrh_score))**

**说明：**贷后融合分对数化，来源brm_ent_dm_work.snap_marm_model_record的v3010314出参，model_id=ADHRCRSS。取log(dhrh_score/(1-dhrh_score))

**04 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.intedate：应还账单日**

**说明：**贷后融合分对数化，来源brm_ent_dm_work.snap_marm_model_record的v3021010入参，model_id=ADHRCRSS。

**05 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.putoutdate_min：客户最早发放借据日期**

**说明：**来源集市借据表brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds，取用户min(putout_date)

**06 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.pd_value：PD值**

```Plaintext
逻辑如下：-- intedate_date：intedate应还账单日yyyy-MM-dd格式
case when overduedays_max > 0 then 1/(1+exp(-0.30939087-ln_dhrh_score*0.73773968))
         when datediff(intedate_date,substr(putoutdate_min,1,10)) >= 541        then 1/(1+exp(1.20581381-ln_dhrh_score*0.7267306))
         else 1/(1+exp(0.61009258-ln_dhrh_score*0.88998344))
         end as pd_value
```

**07 brm_ent_dm_work.dwd_adhrcrss_zhubiaochi_ds.pd_flag：PD类型 0 逾期组 1短客组 2长客组**

```Plaintext
逻辑如下：-- intedate_date：intedate应还账单日yyyy-MM-dd格式
case when overduedays_max > 0 then 0
         when datediff(intedate_date,substr(putoutdate_min,1,10)) >= 541 then 2
         else 1 end as pd_flag        
```
