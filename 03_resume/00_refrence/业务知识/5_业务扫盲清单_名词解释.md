## 1 名词解释

<img src="C:\Users\m8705\AppData\Roaming\Typora\typora-user-images\image-20260214122805754.png" alt="image-20260214122805754" style="zoom:200%;" />

**管理贷款（LUM，Loans under Management）：**微众银行负责持续管理的贷款，包括银保监口径下各项贷款（表内）、联合贷合作方份额贷款、技术性出表资产及其他基于贷款服务合作获取相关费用的贷款。**持续管理**指持续负责或参与贷款的贷前贷中贷后等主要环节，包括但不限于：产品设计、功能开发、营销获客、风控授信、清算结算、账务处理、客服催收等，以及资产流转的发行和存续期管理等。

**表内贷款：**简单概括为未进行出表操作（核销、ABS债权转让、ABS不良收益权转让、代偿、债转等）的我行份额贷款资产。

**出表ABS：**技术性出表资产，包括整笔借据ABS和联合贷我行自持份额ABS等；

**出表ABN：**属于其他基于贷款服务合作获得相关费用的贷款，即我行仅提供贷款通道，实际贷款由合作方出资提供；

**自营债权转让：**已于2023年5月实际开展相关业务；

**不良资产****转让：**第一包资产已于2022年7月封包，2022年9月完成交割；第二包资产第二包于2023年10月封包，2023年12月完成交割；

**分流贷款：**主要为小微银银2.0和小微银银2.1的联合贷划分为我行自营额度的部分贷款。

**小微银银：**我行和合作行以一定比例出资放款的贷款方式（联合贷）；

1. 小微银银2.0：额度模式，额度切分的部分额度分流给合作行（借据信息存储在企贷云系统，集市表打标 source_tbl = 'xwyy'）；
2. 小微银银2.1：借据模式，企金模型额度会切分自营额度和合作行模型额度（我行和合作行分别出资，借据信息存储在mcfcm系统，其中华能信托在我行的出资比例为0）；

## 3 不同口径下的核销后时点余额的加工逻辑

**01 LUM口径-核销后余额计算**

（1）零元回购（nvl(is_lyhg,0)=1）、视同核销（nvl(is_sthx,0)=1）、核销（duestatus in ('3')）、已交割（需要关联不良资产转让表）的借据不计入余额统计范围

（2）发生担保代偿、保险代偿的借据，只统计我行的代偿金额（3）其余可直接取借据表的余额字段（balance）**注：小微银银2.1的借据如果发生不良资产转让，则整笔借据都不计入LUM口径计算**其中，不良资产转让集市表：brm_ent_dm_work.dwd_duebill_blzczr_ds（status = '4' 表示已转让交割）

**01 表内口径-核销后时点余额计算**

（1）首先筛选如下借据（when coalesce(is_outside_table_package,0)=0 and source_tbl not in ('xwyy')）

（2）零元回购（nvl(is_lyhg,0)=1）、视同核销（nvl(is_sthx,0)=1）、核销（duestatus in ('3')）不计入余额统计范围

（3）小微银银2.1联合贷的借据，只统计我行的余额

（4）发生担保代偿、保险代偿的借据，只统计我行的代偿金额

（5）其余可直接取借据表的余额字段（balance）

```SQL
    case when coalesce(is_outside_table_package,0)=0 and source_tbl not in ('xwyy')then (case when nvl(t.is_sthx,0)=1 then 0          
     when nvl(t.is_lyhg,0)=1 then 0           
     when t.duestatus in ('3') then 0           
     when nvl(t.is_xwyy_21,0)=1 then we_balance_xwyy_21              
     when nvl(t.is_bxdc,0)=1 then we_balance_bxdc                 
     when nvl(t.is_dbdc,0)=1 then we_balance_dbdc             
     when nvl(t.is_xwyy_30,0)=1 then we_balance_xwyy_30           
 else t.balance end) else 0 end as balance_bn  --表内核销后余额
```
