# 入催、治愈、还款率

## 1 业务含义

***入催：***客户未在还款日内按时还款，则此笔借据在正常还款日后进入催收环节；客户可以多次入催，但入催后的客户若一直没有还款，到下一个账单日时仍未还款，则在下一个账单日后，此笔借据不认为是被入催的借据；举例：客户共借了12000元，每月的还款日为3号，还款金额为1000，共还12期，2023年12月3开始还款，第一期（12月3日）按时还款；但客户在1月3日没有还款，则客户在1月3日起入催，入催金额为11000元且一直到2月3日，客户仍未还款，但此时的客户不算**再次**入催；

***出催：***客户虽没在还款日按时还款（入催），但在一定周期内还完当前到期的应还的欠款了；举例：接着入催的例子，客户到2月5日还款了，还了1月3日和2月3日的应还余额（可能还有利息），则这个客户此时出催了，出催日为2月5日；共入催 31 + 2 = 33天，入催33天后回款的贷款金额为2000元；

***相关指标：***

• **入催率：**本月入催的余额占上月末正常未逾期的在贷余额（M0）的比例；• **0~N天治愈率：**随着催收策略、手段等变动，使入催的客户在一定催收周期后还款的金额（有效催回的金额）为治愈金额，则治愈率为治愈金额占入催总金额的比例；• **0~N天还款率：**还款金额主要分为：客户在还款日内按时还款的金额和在一定周期内有效催回的金额；前者占比计算为1 - 入催率（客户按时还款的金额比例），后者计算占比为入催率 × 0~N天治愈率（客户未按时还款，但在一定周期内被有效催回的金额比例）；• 对于贷后团队而言，目标是降低入催率（逾期率）,提高还款率，针对性采取催收手段，提高治愈率和回款率；一般前催用户（逾期1-30天）被催回的概率较大，坏账减少，随着逾期天数增加，逾期天数越大的治愈率越低；

| 指标        | 分子                              | 分母              |
| ----------- | --------------------------------- | ----------------- |
| 入催率      | 入催总贷款金额                    | 上月末M0贷款金额  |
| 0~N天治愈率 | 入催N天后回款的贷款金额           | 入催N天总贷款金额 |
| 0~N天还款率 | 1 - 入催率 + 入催率 × 0~N天治愈率 |                   |

## 2 计算流程

根据上述公式和定义，可确定指标计算需获取单笔借据在未来N天的状态，进而比对求算，则

***Step1：计算同笔借据前一天的逾期状态及三天后的在贷余额，逾期天数及日期；***

```Plaintext
,first_value(balance) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as balance_after3_for_last --后3日贷款余额（第3天为空则取之前的）
,first_value(overduedays) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as overduedays_after3_for_last
,first_value(ds) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as ds_after3_for_last
,lag(overduedays,1,null) over (partition by duebill_serialno order by ds) as overduedays_before1 -- 前一日逾期天数
```

***Step2：客户在不同观察时点下的在贷总金额，当前最大逾期天数，未来第3日的在贷总金额、第3日的最大逾期天数，观察时点前一天的最大逾期天数，客户随观察时点变化的1-3天的逾期天数；***

```Plaintext
,sum(balance) as balance --当前在贷总金额
,max(overduedays) as overduedays  --当前最大逾期天数
,sum(balance_after3_for_last) as balance_after3_for_last  --未来第三天的在贷总金额
,max(overduedays_after3_for_last) as overduedays_after3_for_last  --未来第三天的最大逾期天数
,min(ds_after3_for_last) as ds_after3_for_last --未来的第三天日期
,max(overduedays_before1) as overduedays_before1  --前一天的逾期天数
,min(coalesce(overduedays,0)) over (partition by ccif order by ds rows between 1 following and 3 following) as next_3_overduedays --客户随观察时点变化的1-3天的逾期天数
```

***Step3：标记是否为入催客户、第三天是否为有效催收回款客户、催收3天的未还金额；***

1. 若前一时点的逾期天数为0（正常），当前时刻的逾期天数大于0（逾期），说明该客户入催【一般出现在账单日】；

```Plaintext
case when t.overduedays_before1 = 0 and t.overduedays > 0 then 1 else 0 end as is_incall
```

1. 若前一时点的逾期天数为0（正常），当前时刻的逾期天数大于0（逾期），三天后的逾期天数为0（恢复正常），说明该客户为有效催收回款客户，客户三天后出催；

```Plaintext
case when t1.next_3_overduedays = 0 and t.overduedays_before1 = 0 and t.overduedays > 0 then 1 else 0 end as is_cure_3_days
```

1. 若三天后的逾期天数小于当前逾期天数+3，说明该客户为有效回款客户，还款金额为0，反之该客户的未还金额仍为入催第三天的在贷金额；

```Plaintext
case when t.overduedays_after3_for_last < t.overduedays + datediff(from_unixtime(unix_timestamp(t.ds_after3_for_last,'yyyyMMdd'),'yyyy-MM-dd'),from_unixtime(unix_timestamp(t.ds,'yyyyMMdd'),'yyyy-MM-dd'))
                then 0 else t.balance_after3_for_last end as not_pay_back_balance_after3_for_last
```

***Step4：计算入催金额、入催回款成功金额、1-3天回款金额；***

```Plaintext
,sum(case when is_incall = 1 then balance else 0 end) as incall_balance --入催金额
,sum(case when is_incall = 1 and datediff('${run_date_std}',ds_std_origin) >= 3 then balance else 0 end) as incall_balance_for_cure_3_days        --入催3天金额（金额0-3天治愈率分子）
,sum(case when is_incall = 1 and is_cure_3_days = 1 and datediff('${run_date_std}',ds_std_origin) >= 3 then balance else 0 end) as cure_3_days_balance       --入催3天内回款借贷金额（金额0-3天治愈率分母）
,sum(case when is_incall = 1 then balance - not_pay_back_balance_after3_for_last else 0 end) as pay_back_balance_1_3                                          --1-3天回款金额
--计算m0贷款金额和m0贷款客户数，便于计算入催率、还款率
,count(case when overduedays = 0 then ccif else null end) as m0_cnt
,sum(case when overduedays = 0 then balance else 0 end) as m0_balance
```

***Step5：计算入催率、0~3天治愈率、1~3天回款率、0~3天还款率；***

```Plaintext
,incall_balance / m0_balance as incall_balance_rate   --入催率
,cure_3_days_balance / incall_balance_for_cure_3_days as cure_3_days_balance_rate --0-3天治愈率
,pay_back_balance_1_3 / incall_balance as pay_back_balance_1_3_rate --1-3天回款率
,1 - incall_balance / m0_balance + (incall_balance / m0_balance * cure_3_days_balance / incall_balance_for_cure_3_days) as pay_back_balance_rate_0_3 --0-3天还款率 1-a+ab
```

## 3 参考代码

```SQL
--Step2：因此可获取客户在不同观察时点下的在贷总金额，当前最大逾期天数，未来第3日的在贷总金额、第3日的最大逾期天数，观察时点前一天的最大逾期天数，客户随观察时点变化的1-3天的逾期天数；
with tmp as ( --以ds和客户维度计算当前贷款金额、当前最大逾期天数、未来第3天的贷款金额、未来第3天的最大逾期天数、第三日的日期、前一天的最大逾期天数
    select ds,ccif
            ,sum(balance) as balance
            ,max(overduedays) as overduedays
            ,sum(balance_after3_for_last) as balance_after3_for_last
            ,max(overduedays_after3_for_last) as overduedays_after3_for_last
            ,min(ds_after3_for_last) as ds_after3_for_last
            ,max(overduedays_before1) as overduedays_before1
    from(--Step1：计算同笔借据昨天的逾期状态及三天后的在贷余额，逾期天数及日期；
        select *
            ,first_value(balance) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as balance_after3_for_last --后三日的贷款余额
            ,first_value(overduedays) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as overduedays_after3_for_last --后三天的逾期状态
            ,first_value(ds) over (partition by duebill_serialno order by ds desc rows between 3 preceding and current row) as ds_after3_for_last --后三天的观察时点
            ,lag(overduedays,1,null) over (partition by duebill_serialno order by ds) as overduedays_before1 --前一日逾期天数
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds >= '20230601' 
    )
    group by ds,ccif
),temp as (--以ds和客户维度计算3天内逾期天数
    select ds,ccif,min(coalesce(overduedays,0)) over (partition by ccif order by ds rows between 1 following and 3 following) as next_3_overduedays
    from(
        select ds,ccif,max(overduedays) as overduedays
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds >= '20230601' and duestatus in ('0','1','3') and source_tbl in ('mcfcm','ymcfcm','cfcm') and is_oneyidai <> 1
        group by ds,ccif
    )
)--Step4：计算入催金额、入催回款成功金额、1-3天回款金额；
select ds_month,
    incall_balance / m0_balance as incall_balance_rate,   --入催率
    cure_3_days_balance / incall_balance_for_cure_3_days as cure_3_days_balance_rate , --0-3天治愈率
    pay_back_balance_1_3 / incall_balance as pay_back_balance_1_3_rate, --1-3天回款率
    1 - incall_balance / m0_balance + (incall_balance / m0_balance * cure_3_days_balance / incall_balance_for_cure_3_days) as pay_back_balance_rate_0_3 --0-3天还款率 1-a+ab
from(
    select substr(ds,1,6) as ds_month
        ,sum(case when is_incall = 1 then balance else 0 end) as incall_balance --入催金额
        ,sum(case when is_incall = 1 and datediff('${run_date_std}',ds_std_origin) >= 3 then balance else 0 end) as incall_balance_for_cure_3_days  --入催3天金额（金额0-3天治愈率分子）
        ,sum(case when is_incall = 1 and is_cure_3_days = 1 and datediff('${run_date_std}',ds_std_origin) >= 3 then balance else 0 end) as cure_3_days_balance --入催3天内回款借贷金额（金额0-3天治愈率分母）
        ,sum(case when is_incall = 1 then balance - not_pay_back_balance_after3_for_last else 0 end) as pay_back_balance_1_3 --1-3天回款金额
    from(--Step3：标记是否为入催客户、第三天是否为有效催收回款客户、催收3天的未还金额；
        select t.ds,t.ccif,t.overduedays,t.balance
                ,concat(substr(t.ds,1,4),'-',substr(t.ds,5,2),'-',substr(t.ds,7,2)) as ds_std_origin
                ,case when t.overduedays_before1 = 0 and t.overduedays > 0 then 1 else 0 end as is_incall
                ,case when t1.next_3_overduedays = 0 and t.overduedays_before1 = 0 and t.overduedays > 0 then 1 else 0 end as is_cure_3_days
                ,case when t.overduedays_after3_for_last < t.overduedays + datediff(from_unixtime(unix_timestamp(t.ds_after3_for_last,'yyyyMMdd'),'yyyy-MM-dd'),from_unixtime(unix_timestamp(t.ds,'yyyyMMdd'),'yyyy-MM-dd'))
                      then 0 else t.balance_after3_for_last end as not_pay_back_balance_after3_for_last
        from tmp t
        left join temp t1 on t.ds = t1.ds and t.ccif = t1.ccif
    )
    group by substr(ds,1,6)
)t
left join(--计算m0贷款金额和m0贷款客户数，便于计算入催率、还款率
    select ds
        ,count(case when overduedays = 0 then ccif else null end) as m0_cnt
        ,sum(case when overduedays = 0 then balance else 0 end) as m0_balance
    from (
        select ds,ccif,max(overduedays) as overduedays,sum(balance) as balance
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds >= '20230601' and duestatus in ('0','1')
            and substr(date_add(from_unixtime(unix_timestamp(ds,'yyyyMMdd'),'yyyy-MM-dd'),1),9,2) = '01' --月末
            and months_between('${run_date_std}',from_unixtime(unix_timestamp(ds,'yyyyMMdd'),'yyyy-MM-dd')) <= 1
        group by 1,2
    )
    group by ds
)t0 on replace(substr(date_add(from_unixtime(unix_timestamp(t0.ds,'yyyyMMdd'),'yyyy-MM-dd'),1),1,7),'-','') = t.ds_month
;
```
