# 另类年化风险成本率

## 1 业务含义

***另类年化风险成本率：***我行根据不良资产的转移周期（约为三个月）进一步预测未来三个月的不良资产变化情况；日均贷款余额的变化幅度较小，因此另类年化风险成本率与年化风险成本率相差较小；

***另类年化风险成本率***常用于直观分析净不良新增资产的质量变化（已发放借据的质量变化），更能暴露客户的风险水平和窗口转变情况，可观察不同产品的客群的风险暴露情况；

**【分母】：T月日均正常在贷M0余额【借据状态为正常在贷且未逾期】;**

**【分子】：T+3月核销前新增不良余额：**观察同笔借据在T+2月月末时点的借据状态到T+3月月末时点的不良金额的变化；

**➤** ***核销前新增不良余额-财务口径【可参考《风险成本率-借据维度》的文档】***

T月核销前新增不良余额【C】 = T月不良新发生【A】- T月不良回退及清收【B】，其中，余额均进行不良资产转让还原；以借据粒度观察不良金额变化可能存在如下几种情况：以月末为观察时点（1）借据状态：T-1月为未核销（非'3'）或非不良('01','02')，T月为核销（'3'）或不良('03','04','05') 即一般新增不良（2）借据状态：T-1月为核销（'3'）或不良('03','04','05')，T月为核销（'3'）或不良('03','04','05') 即存量不良_当前不良（3）借据状态：T-1月为核销（'3'）或不良('03','04','05')，T月为未核销（非'3'）或非不良('01','02') 即存量不良_当前非不良• **T月不良新发生余额【A】为第（1）种情况的T月余额之和；**• **T月不良回收及清退余额【B】为第（2）+（3）种情况的回退余额之和，其中第（2）种的回退余额为T月的不良余额与T-1月的不良余额的差，第（3）种回退余额为T-1月的不良余额；**

## 3 另类风险成本率-中间表

### 3.1 企金整体-另类风险成本率-中间表-介绍

• **dw1713-中间表：brm_ent_dm_work.dwd_fxcb_yqlv_by_duebill_ds_extend** • **粒度：**借据号+观察时点(天)【duebill_serialno + watching_ds】 • **更新：**月末更新 **➤** ***中间表-加工逻辑：***

☑ 整体量级取至借据表【粒度：借据号，范围：2018年12月31日起】；☑ 剔除万一贷的借据；☑ 剔除冲销的借据；☑ 处理方法：不含新借据，仅保留最原始借据（如借新还旧、无还本续贷）；

**➤** ***中间表-余额加工口径：***

☑ 核销前新增不良余额：借据的五级分类取不良资产转让处理后的五级分类 

 【1】LUM口径：保险代偿、担保代偿、不良资产转让均纳入核销前新增不良余额统计；

☑ 日均在贷正常余额：借据的逾期天数、五级分类、状态取不良资产转让处理后的情况  

 【1】LUM口径：保险代偿、担保代偿、不良资产转让均纳入日均在贷正常余额统计；

**➤** ***中间表-数据结构-注释：***

（1）针对小集合量级，建议保留借据所有观察时点的情况；多对一的情况是否考虑（影响较小）；（2）借据冲销可能导致上个月有这笔借据，当月没有的情况；（3）2023年10月14日调整了借据表表内标签（is_outside_table_package字段）的加工逻辑，导致dw1713中间表加工出来的金额与2023年9月前的报表数值存在一点差异（主要在于日均贷款值存在差异）；（4）中间表的核销新增不良余额的计算逻辑是对齐财务口径的（即考虑核销借据的余额变化【核销净额】）； (5) 另类风险成本率的分子和分母至差两个月，集市表的加工已错位关联至分母月，因此使用集市表无需再进行错位关联，直接借鉴参考代码使用即可；

### 3.3 中间表-使用方法

```Plaintext
--当月新增不良资产：sum(balance_bl_m_3)
--举例：当月新增不良资产：sum(balance_bl_m_3)
--举例：另类风险成本率-日均贷款：sum(balance_bl_m0)/count(distinct watching_ds)
select ds_month,
       sum(coalesce(balance_bl_m_3,0))/100000000 as buliang_add_lum,
       sum(coalesce(balance_bl_m0,0))/count(distinct watching_ds) as avg_balance_lum,
       sum(coalesce(balance_bl_m_3,0))/(sum(coalesce(balance_bl_m0,0))/count(distinct watching_ds))*12 as fxcb_rate_lum,
       sum(coalesce(balance_bn_bl_m_3,0))/100000000 as buliang_add_bn,
       sum(coalesce(balance_bn_bl_m0,0))/count(distinct watching_ds) as avg_balance_bn,
       sum(coalesce(balance_bn_bl_m_3,0))/(sum(coalesce(balance_bn_bl_m0,0))/count(distinct watching_ds))*12 as fxcb_rate_bn     
from brm_ent_dm_work.dwd_fxcb_yqlv_by_duebill_ds_extend
where ds = '20231231'-- 添加行业/客群/企金整体等筛选条件
group by ds_month
order by ds_month
```
