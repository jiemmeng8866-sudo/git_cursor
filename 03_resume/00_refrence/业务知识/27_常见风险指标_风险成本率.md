# 风险成本率

## 1 业务含义

***风险成本率（新增******不良******率）:***某一时间段内新增的不良金额【分子】占该时间段内的日均在贷余额【分母】的比重，若风险成本率指标升高，其反映观察时点内转变成不良的借据金额变多，对应的风险成本增高，若风险成本率达到设定的阈值，则说明需采取手段降低风险，因此，风险成本率是信贷中重要的分析指标；

**➤** ***【分母】核销后日均******在贷******余额******：***与客户贷款产品的还款计划及客户是否正常按期还款的行为均有关；

在观察时间范围内的平均每日的在贷余额；（不考虑已被不良核销处置等的借据）

**➤** ***【分子】核销前新增******不良******余额******：***借据在观察时点下的变化状态（一般分为不良新发生和不良回退两种情况）；

**【****不良****新发生】:**相较于前某个时点下，新转变为不良的借据（举例：同笔借据的在T-1月为非不良状态，但在T月转变成不良的状态）；**【不良回退】:**相较于前某个时点下，原不良借据的余额发生回退（举例：同笔借据在T-1月为不良但在T月转变为非不良的状态、同笔借据在T-1月为不良，但在T月不良余额减少的情况）；**新口径下，新增不良不仅仅包含从不良变为非不良状态的借据余额变化情况，还包含被核销处置的借据的核销余额的变化情况【两部分】**

**➤** ***核销前新增不良余额-计算方法***

**【整体维度】：**直接计算每个月的不良资产总余额，则本月的核销前新增不良资产 = 本月月末的总不良资产 - 上月月末的总不良资产；**【借据维度】：**从借据维度观察同笔借据从上个月月末到本月月末的不良金额变化情况（不良新发生和不良清收与回退），因此新增不良资产=不良新发生的借据金额-不良回退的借据金额；

**（2）vintage类-风险成本率=月末时点核销前不良贷款余额/观察时点到基准时点的时间范畴下的日均在贷余额** 【1】发放时点-风险成本率： 借据自**发放时点**起至观察时点内的日均在贷余额； 【2】首授时点-风险成本率： 借据自**首次授信时点**起至观察时点内的日均在贷余额； 【3】定期核额时点-风险成本率： 借据**自定期核额时点**起至观察时点内的日均在贷余额； 【4】最新授信时点-风险成本率： 借据自**最新授信时点**起至观察时点内的日均在贷余额；

## 2 不良资产转让还原处理的原因及逻辑

**其一，**由于不良资产转让处置操作会导致被处置的借据的状态（借据表的余额、状态）会发生改变，若仍只通过五级分类来计算观察时点的不良资产，可能会出现明显的数据波动，理应将此部分借据均还原至转让交割前的借据状态（借据余额、逾期天数、五级分类）；

**其二，**表内资产和表外资产均可能进行不良资产转让处置操作，则均需纳入借据的还原逻辑内；

**其三，**自然月口径-风险成本率和另类风险成本率的新增不良分子主要是考虑封包转让时点前后的借据变化，而vintage类风险成本率主要考虑转让时点前后的借据变化；

**其四，**以上涉及不良资产转让还原的指标有风险成本率、另类风险成本率、发放时点-不良-vintage、m1转m2、m1转不良、m2转不良占比、回收率等指标；此外，微众易贷产品不存在/无需不良资产转让还原处理；

***01 借据的时点余额***

【1】封包转让后的表内借据：若借据为小微银银2.1的借据，则还原金额取package_amt - transfer_discount_amt + un_balance_xwyy_21(封包金额 - 折价金额 + 合作行余额)     反之，则还原的金额取package_amt - transfer_discount_amt(封包金额 - 折价金额)【2】封包未转让的表内借据：若借据为小微银银2.1的借据，则还原金额为package_amt + un_balance_xwyy_21      反之，则还原的金额为package_amt【3】交割后的表外借据，则借据还原的金额为借据交割前一天的借据余额【4】其他：直接取时点余额balance

***02 借据的时点逾期天数***

【1】借据在封包前的逾期天数直接取当时时点的逾期天数overduedays 【2】表内借据在封包后的逾期天数取封包时点前一天的逾期天数：package_overduedays_before【3】表外借据在交割后的逾期天数取交割时点前一天的借据逾期天数

***03 借据的五级分类***

【1】借据在封包前的五级分类直接取当时时点的五级分类classifyresult 【2】表内借据在封包后的五级分类为了方便处理，直接设为'05'【3】表外借据在交割后的五级分类取交割前一天的借据五级分类状态（一般为'05'）

**【备注：表内的借据转让后的is_outside_table_package标签会变成1，所以要进行还原处理】**

***Step1 - 不良资产转让还原处理：将已封包/转让处理的不良借据均还原至交割转让前一天的借据状态；***

***Step2 - 核销前新增不良余额和核销后日均在贷余额的计算：***

**➤** ***核销前新增不良余额***

一般来说，T月核销前新增不良余额 = T月核销前总不良贷款余额 - T-1月核销前总不良贷款余额（此处仍保留此基准计算逻辑），即 9月核销前新增不良余额 = 9月月末时点总不良贷款余额 - 8月月末时点总不良贷款余额**（注：此处的余额均已进行不良资产转让还原）；**

但我行财务口径下的新增不良余额 = 核销后新增不良余额（核销后时点不良余额之差） + 观察时间范畴内的核销净额 + 不良资产转让余额，即 9月核销前新增不良余额 = 9月月末核销后时点不良余额 - 8月月末时点核销后时点不良余额 + 9月内核销净额 + 9月内不良资产转让的余额；即 9月核销前新增不良余额 = 9月月末时点核销或不良的总余额 - 8月月末时点核销或不良的总余额**（注：此处的余额均已进行不良资产转让还原）；**

**注意：集市表加工-自然月风险成本率的口径对齐财务口径**

**➤** ***核销后日均在贷余额***

【1】时点的核销后在贷余额需限定借据状态为非核销，即duestatus in ('0','1')；【2】以月份为计算维度来计算当月核销后日均在贷余额：当月核销后的总在贷余额/当月天数；

***Step3 - 年化风险成本率的计算***

（1）常用口径 - T月年化风险成本率 = T月核销前新增不良余额 / T月核销后日均在贷余额 × 12（2）财务口径 - T月年化风险成本率 = T月核销前新增不良余额 / T月核销后日均在贷余额 × （当年的天数/T月的天数）

**借据维度的计算难点在于客户进行借新还旧所新生成的借据本质是结清旧借据，发放新借据，而基于不良新发生和不良回退的定义，则借新还旧产生的新借据应还原至旧借据，从旧借据的维度来观察借据的不良变化情况，因此如何关联旧借据与新借据是计算口径的重要难点，此处仅介绍以借据维度如何计算核销前新增不良资产，不再介绍日均贷款余额的计算（具体参考整体维度计算口径）。**

***Step1 - 不良资产转让还原处理：将已封包/转让处理的不良借据均还原至交割转让前一天的借据状态；***

![image-20260214130459498](C:\Users\m8705\AppData\Roaming\Typora\typora-user-images\image-20260214130459498.png)



***Step2 - 借据重组：***将全部借据还原重组保留旧借据字段，若发生过借新还旧的新借据会匹配至最早一笔非万一贷的旧借据，保留最早一笔的旧借据号；未发生借新还旧的借据，保留当前的借据号；

***Step3 - 合理映射：***将新借据的最新状态映射回旧借据，并仅保留发放时间最新的一笔借据信息，确保不会重复计算；【历史存在多对一的重组操作，其实按比例还原借据的金额会更准确一些】

***Step4 - 以借据维度观察借据在T-1月至T月的借据变化情况：***

***不良新发生和不良清收与回退：***

借据维度计算新增不良：T月核销前新增不良余额 = T月不良新发生余额 - T月不良回退余额

**常用口径：需与整体维度计算新增不良余额值对齐【不良新发生】:**相较于前某个时点下，新转变为不良的借据（举例：同笔借据的在T-1月为非不良状态，但在T月转变成不良的状态）；**【不良回退】:**相较于前某个时点下，原不良借据的余额发生回退（举例：同笔借据在T-1月为不良但在T月转变为非不良的状态、同笔借据在T-1月为不良，但在T月不良余额减少的情况）；

**财务口径：需与整体维度计算新增不良余额值对齐【不良新发生】：**借据在T-1月月末的五级分类为非不良且状态为非核销，在T月月末的五级分类变为不良或状态为核销，则新增不良金额为T月月末的余额；**【不良回退】：**（1）借据在T-1月月末的五级分类为不良或状态为核销，在T月月末的五级分类仍为不良或状态仍为核销，则新增不良金额为月末时点余额的变化额；（2）借据在T-1月月末的五级分类为不良或状态为核销，在T月月末的五级分类为非不良且状态为非核销，则新增不良金额为T-1月月末的余额；

注：实际上从借据维度和整体维度所计算新增不良余额值存在一定的差异，因为存在新发放借据（非纾困操作）在发放之后基于各种上线规则（例如破产）的五级分类变成不良（一般借据在逾期超60+，五级分类才会是不良），此类借据情况较少，但未来随着规则上线，会逐渐增加。

        ,case when ((e.duebill_serialno is not null and e.package_date <= t.ds and e.transfer_date <= t.ds and e.status=4) or coalesce(is_outside_table_package,0) = 0)
              and source_tbl in ('mcfcm','cfcm','ymcfcm')
              then(case when nvl(t.is_sthx,0)=1 then 0
                        when nvl(t.is_lyhg,0)=1 then 0
                        when e.duebill_serialno is not null and e.package_date <= t.ds and e.transfer_date <= t.ds and e.status=4 then e.package_amt - e.transfer_discount_amt           --转让后 
                        when e.duebill_serialno is not null and e.package_date <= t.ds and (e.transfer_date > t.ds or e.transfer_date is null) and e.status in (3,4) then e.package_amt   --封包后转让前                                           
                        when nvl(t.is_xwyy_21,0)=1 then we_balance_xwyy_21
                        when nvl(t.is_bxdc,0)=1 then we_balance_bxdc       
                        when nvl(t.is_dbdc,0)=1 then we_balance_dbdc   
                        when zj.serialno is not null and t.balance > zj.discountbalance then t.balance - zj.discountbalance
                        when zj.serialno is not null and t.balance = zj.discountbalance then 0
                        when zj.serialno is not null and t.balance < zj.discountbalance then 0
                        else t.balance end)
            else 0 end as balance_bn_blzczr_zj  --表内   新增不良 分子

        ,case when e.duebill_serialno is not null and e.package_date <= t.ds then e.package_overduedays_before 
              when d.duebill_serialno is not null and d.trans_date <= t.ds then d.overduedays
              else t.overduedays end as overduedays_blzczr  --逾期天数

        ,case when e.duebill_serialno is not null and e.package_date <= t.ds then '05' 
              when d.duebill_serialno is not null and d.trans_date <= t.ds then d.classifyresult
              else t.classifyresult end as classifyresult_blzczr  --五级分类

        ,case when e.duebill_serialno is not null and e.package_date <= t.ds then e.duestatus_before
              when d.duebill_serialno is not null and d.trans_date <= t.ds then d.duestatus
              else t.duestatus end as duestatus_blzczr --借据状态
    from(--借据表--
        select *
        from brm_ent_dm_work.dwd_mcfcm_business_duebill_etl_ds
        where ds = '${run_date}' and length(ds) = 8 --and ds >= '20230101'
    ) t

    left join(--不良资产转让--
        select duebill_serialno
            ,package_amt
            ,replace(substr(package_date,1,10),'-','') as package_date
            ,replace(substr(transfer_date,1,10),'-','') as transfer_date
            ,package_overduedays_before
            ,transfer_discount_amt
            ,duestatus_before
            ,status
        from brm_ent_dm_work.dwd_duebill_blzczr_ds t
        where t.ds='${run_date}' and t.status in ('3','4')
    ) e on t.duebill_serialno = e.duebill_serialno

    left join(--折价回购处理--
        select *
        from brm_wyd_ods_mask.ods_mcfcm_acct_loan_etl_bdcn
        where repurchasetype = '1'
        and discountbalance is not null 
        and ds = '${run_date}'
    ) zj on zj.ds = t.ds and t.duebill_serialno = zj.serialno

    left join(--表外资产转让--
        select duebill_serialno
               ,balance
               ,classifyresult
               ,overduedays
               ,duestatus
               ,replace(substr(transdate,1,10),'-','') as trans_date
        from brm_ent_dm_work.dwd_duebill_bw_blzczr_20240131
    )d on t.duebill_serialno = d.duebill_serialno
)t
;
