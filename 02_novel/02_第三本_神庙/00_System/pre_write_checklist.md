# 写前强制检查点 (Pre-Write Checklist)

> **定位**：每次写新章节前，必须逐条完成以下检查。未完成检查前，禁止写入正文。此文件与 `Anti_Repetition.md` 配合使用——前者防剧情重复，本文件防框架脱轨。

---

## 一、框架对齐检查（Framework Alignment）

### 1.1 章节位置确认

回答以下问题：

```
本章计划编号：Ch_____
所属卷：第___卷 
所属阶段（查阅对应Vol_Outline）：___
框架规定本章段的核心事件（查阅 `02_Outline/00_唯一大纲.md`）：___
```

### 1.2 框架消费状态

> 唯一大纲中已无"计划章节号"概念——大纲只描述实际发生的事件。检查本章是否与已完成事件冲突即可。

### 1.3 地图位置一致性

```
上一章结尾位置：___
本章场景位置：___
地理移动是否合理？___
```

## 六、框架脱轨熔断机制（Circuit Breaker）

以下任一条件触发时，**禁止写正文**，必须输出错误报告：

### 触发条件

1. **先决事件未完成**：框架规定A必须在B之前发生，但B已被写入而A未写入 → **熔断**
2. **地图跨越**：角色在没有传送阵/合理移动手段的情况下跨越大半个大陆 → **熔断**（除非有明确传送）
3. **女神提前唤醒**：某女神在当前框架阶段尚不具备唤醒条件但已写入唤醒 → **熔断**
4. **关键道具跳过**：框架规定的关键道具（如极阳之火、星辰泪）尚未获取但相关剧情已跳过 → **熔断**
5. **好感度跳跃**：女神好感度从-10直接跳到+50，中间缺少关键事件 → **熔断**

### 熔断输出格式

```
[FRAMEWORK CIRCUIT BREAKER - 框架脱轨熔断]

触发条件：#___ 
框架规定：___
实际写作状态：___
缺失的先决条件：___
建议修复方案：
  A. ___
  B. ___
请选择修复方案或手动覆盖熔断。
```

## 八、快速自检卡（Quick Self-Check）

每章写完后，用30秒回答：

1. 查阅 `00_唯一大纲.md`：框架规定本章段该发生什么？实际写了什么？二者是否匹配？
2. 是否有框架规定必须先发生的剧情被跳过了？
3. 角色当前位置/状态是否与上一章连续？
4. 新埋的伏笔是否已在Event_Log登记？
5. 本章是否与已有章节有相似的场景节拍？（参照Anti_Repetition.md）

5个问题全部通过 → 章节合格。
任一问题不通过 → 回到对应步骤修复。
