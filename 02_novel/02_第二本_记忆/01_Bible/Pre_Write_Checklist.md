# 预写检查清单

> **用途**：每章写作前强制执行。防止 AI 上下文衰减导致的情节重复、角色偏离、设定矛盾。
> **使用时机**：打开空白章节文件之前。

---

## 五步检查（必须逐项通过）

### 第一步：定位当前弧段

- [ ] 打开 `02_Outline/Main_Plot.md`，确认当前写到哪个弧段、弧段主题是什么
- [ ] 打开当前卷的 `02_Outline/VolX_Outline.md`，确认本章在弧段中的位置
- [ ] 确认上一章结尾的角色位置和状态（防止跳线）

- 本章方向：第零代继续往源头深处找下一个标签，传回更高温度

### 第二步：防重复审查

- [ ] 打开 `01_Bible/Event_Log.md`，滚动到当前卷末尾的"已用场景/意象清单"
- [ ] 确认本章计划写的场景类型、情绪节拍、意象不与之重复
- [ ] 检查"已用反噬类型"表——本章如有新客人/新代价，类型不能撞
- [ ] 检查"已用琥珀类型"表——新琥珀的颜色/气味/温度不能撞
- [ ] 打开 `01_Bible/Anti_Repetition.md`，确认高危短语不在本章出现：
  - 陆渊擦拭动作（方巾/麂皮/镜片）
  - 骨瓷茶杯/凉茶
  - 苏离波板糖/咔嚓
  - 霉斑褪/幽蓝光
  - "你可以走了"

**当前卷已用场景**（Ch87）：源头石壁前（无光空间/温度触觉）、码头栈桥（靳明舟面朝东南）、当铺柜台（郑挽发现纸杯导航线）、铜室（钟歧读骸骨）、井口（灰面硬化）、下水道（楚歌检查渊含量）

### 第三步：角色状态校准

- [ ] 确认角色知识——角色不知道其他 POV 线发生的事（除非有信息传递场景）
- [ ] 确认角色语气——陆渊陈述句为主不解释 / 苏离短句 / 楚歌疲惫黑色幽默 / 靳明舟商业术语

## 写完后更新清单

写完一章后，必须更新以下文件：

- [ ] `01_Bible/Event_Log.md`——追加本章条目（核心事件+新设定+场景+伏笔+角色状态）
- [ ] 如果角色状态有重大变化→更新 `01_Bible/Character_Sheets.md` 对应角色
- [ ] 如果引入新设定→更新 `01_Bible/World_Rules.md`
- [ ] 如果推进/回收线索→更新 `02_Outline/Main_Plot.md` 线索状态
- [ ] 如果是卷末章→更新 `01_Bible/Timeline.md`
